# 串口通信一站式 — 项目需求

来源：整合自原始 requirements.md 并允许新增需求与扩展。

## 目标

提供一个能在 Windows 或 Ubuntu 上运行，或作为在线网页版本使用的工具，能够根据用户配置（波特率、数据位、校验位、发送端/接收端语言等）快捷生成串口通信的发送端与接收端解析函数。

## 核心需求

- 支持用户对数据位进行个性化命名。
- 提供便捷的 UI，支持一键选择数据位、数据类型并配置结构体。
- 可生成发送端与接收端对应语言（C、C++、Python）的解析函数。
- 数据包结构体支持 int、float、char（以及可扩展的基本类型），结构体命名可自定义。
- 自动计算并提供数据包总长度（PACKET_SIZE），用于校验、长度前缀或缓冲区分配；长度应根据字段类型与定长字符串长度动态计算。
- 函数命名规范：
  - 发送端编码函数为 encode，输入为已写入数据的结构体，输出为可直接发送的字节流。
  - 接收端解码函数为 decode，输入为接收到的字节流，输出为数据结构体。
  - 发送端的校验函数命名为 send_Verify；接收端的校验函数命名为 recive_Verify。
- 支持多种校验位选择（无校验、奇校验、偶校验、自定义校验算法）。

## 可选/扩展需求（可在 NEW_REQUIREMENT_TEMPLATE.md 新增）

- 支持更多语言（如 Java、Go、Rust）。
- 支持可视化拖拽式结构体编辑器。
- 支持多平台构建脚本与打包（Win exe、Linux bin、Web asm）。
- 示例生成：提供若干典型通信协议示例（电机控制、传感器数据、遥测等）。

## 验收标准

- 能根据 UI 配置生成对应语言的可编译示例代码。
- 发送端生成的 encode 输出能被对应语言的 decode 正确解析回原结构体。
- 校验函数能正确检测并报告数据错误。
